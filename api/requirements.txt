# Slim requirements for the Vercel Python serverless runtime.
#
# Why a separate file from apps/api/requirements.txt:
#   * `weasyprint` needs `libpango` + `libcairo` system libs that Vercel
#     Python doesn't provide. Dropped — PDF report generation degrades
#     to the `reportlab` fallback (already in tree under
#     services.boq_io). Module-level imports of weasyprint must be
#     lazy or guarded.
#   * `llama-index`, `elasticsearch`, `pdfplumber`, `python-docx` —
#     not request-path dependencies. Dropped to stay under the 50MB
#     function-size budget on the Hobby tier. If your demo needs
#     DOCX/PDF parsing, upgrade to Pro and re-add them here.
#   * `sentry-sdk` — kept, useful for catching prod errors.
#   * `celery` — dropped, no worker on Vercel.
