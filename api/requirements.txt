# Slim requirements for the Vercel Python serverless runtime.
#
# Why a separate file from apps/api/requirements.txt:
#   * `weasyprint` needs `libpango` + `libcairo` system libs that Vercel
#     Python doesn't provide. Dropped ‚Äî PDF report generation degrades
#     to the `reportlab` fallback (already in tree under
#     services.boq_io). Module-level imports of weasyprint must be
#     lazy or guarded.
#   * `llama-index`, `elasticsearch`, `pdfplumber`, `python-docx` ‚Äî
#     not request-path dependencies. Dropped to stay under the 250MB
#     unzipped function-size limit.
#   * `sentry-sdk` ‚Äî kept, useful for catching prod errors.
#   * `celery` ‚Äî dropped, no worker on Vercel.
#   * `boto3` / `aioboto3` ‚Äî 80MB combined; both imported only inside
#     function bodies (never at module level), so dropping them is safe.
#     S3 endpoints will raise NameError on cold start in routes that
#     reach the `import boto3` line; those routes (file upload, PDF
#     report storage) will 500 ‚Äî acceptable for a demo where users
#     interact with the dashboard but don't upload large files.
#   * `arq` ‚Äî only `apps/api/workers/queue.py` imports it. Every
#     reference from routers (costpulse line 816, siteeye line 115)
#     is inside a function body. Dropping it doesn't break startup;
#     enqueue_* calls 500 instead. Vercel can't run arq workers
#     anyway (no long-running consumer).
#   * `openai` (raw SDK) ‚Äî drawbridge title-block OCR + bidradar
#     embed_tender use it, both lazy-imported. Dropped ‚Äî those
#     features 500 on call, demo is fine without them.
